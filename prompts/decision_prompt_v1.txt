Decide whether the trading system should buy, sell, hold, or close a position for the next {{timeframe}} bar.

You may only choose one symbol from the allowed list in the snapshot.

Return a JSON object with these fields:
- timestamp
- symbol
- action
- size_pct
- take_profit_pct
- stop_loss_pct
- ttl_minutes
- confidence
- reason
- prompt_version

Formatting rules for numeric ratios:
- `size_pct`, `take_profit_pct`, and `stop_loss_pct` must all be decimal fractions, never percentages.
- Write `0.20` instead of `20`.
- Write `0.007` instead of `0.7`.
- Write `0.0035` instead of `0.35`.
- `ttl_minutes` should usually be one of `5`, `15`, `30`, or `60` for this 5m strategy. Use `0` only for immediate hold/close style decisions.
- Do not use `%`.
- If the snapshot market type is `future`, `size_pct` is the pre-leverage margin fraction of account equity and leverage is applied after that to determine actual notional.
- If `action` is `buy` or `sell`, choose a `size_pct` whose leveraged notional is at least as large as `exchange_min_cost_quote` for that symbol.
- The rule layer is a hard safety floor around max size, exposure, execution feasibility, and protective stops. It is not your directional veto layer, so you should own the entry, hold, and close judgment.
- For futures entries, avoid tiny starter sizes by default. When a fresh setup is genuinely clear after costs, prefer `size_pct` around `0.10` to `0.15` unless the snapshot gives a strong reason to stay smaller.

Decision policy:
- The snapshot uses the latest fully closed {{timeframe}} bar.
- The snapshot indicators are bar-based, not wall-clock aliases:
  - `return_1bar` is the return of the latest closed {{timeframe}} bar.
  - `return_24bars` is the return across 24 closed {{timeframe}} bars.
- A fresh futures entry must clear estimated round-trip cost. Treat `0.12%` as a practical minimum cost hurdle for new entries.
- If `candidate_filter` has already selected up to three symbols, treat them as ranked possibilities rather than as proof that only perfect setups are allowed.
- If the snapshot includes `higher_timeframe.trend_phase`, read it as stage context rather than a trade signal by itself: `trend/reclaim` can support a fresh entry, `pullback` still needs confirmation, and `exhaustion` should make you skeptical of chase entries.
- If `higher_timeframe.model_signal` is present, treat it as a trained 1h regression forecast, not as a hard rule. When its `direction` agrees with the setup and `prediction_strength` is above `1.0`, it is a supportive second opinion; when it sharply conflicts, be more cautious.
- If `candidate_context.setup_phase` is present, treat it as the current 5m setup label. Prefer confirmed phases over `*_unconfirmed` or `*_chase`, and do not let unchanged 1h bias override a weak 5m setup.
- If `candidate_context.traditional_signal_context` is present, treat it as classic chart-structure context rather than as a hard rule. Pattern labels like `failed_rebound_breakdown` or `fresh_support_break` mean a traditional short setup is forming; `failed_breakdown_reclaim` or `fresh_resistance_reclaim` mean a traditional long reclaim is forming.
- If `candidate_context.setup_model_signal` is present, treat it as the narrowest learned signal in the stack: it is trained on this symbol and this `setup_phase` for short-horizon post-cost edge. Positive `predicted_edge_pct` with `label=favorable` and reasonable `sample_count` supports acting; negative `predicted_edge_pct` means default wait unless everything else is exceptional.
- Distinguish `setup_model_signal.quality`: `strong_favorable` means the narrow model is both positive and statistically clearer; `weak_favorable` means the sign is positive but the edge is thin or unstable, so default wait unless the rest of the setup is unusually strong.
- If `candidate_context.entry_viability_preview` is present, treat it as a non-veto diagnostic from the current risk heuristics. A clearly positive `expected_edge.edge_surplus_pct` with empty `shadow_open_signal_reasons` supports acting; a positive `required_threshold_gap_pct` or multiple shadow reasons means waiting is usually better unless the rest of the snapshot is unusually strong.
- If a short candidate has `traditional_signal_context.conviction_score >= 0.55`, `traditional_signal_context.terminal_risk=false`, positive `entry_viability_preview.expected_edge.edge_surplus_pct`, and no `shadow_open_signal_reasons`, do not reject it only because local RSI is already low. That combination can describe a valid failed-rebound breakdown, so a starter short is acceptable unless the bar is clearly climactic.
- If that same short also has `candidate_context.setup_model_signal.quality=strong_favorable`, prefer trusting the setup over a generic “RSI already low” objection unless the bar is obviously terminal.
- If the same short only has `candidate_context.setup_model_signal.quality=weak_favorable`, treat that as insufficient by itself; prefer waiting for one more confirming bar or a cleaner break.
- If `candidate_context.setup_phase=short_breakdown_confirmed`, treat it as a real downside momentum setup rather than a terminal flush. A small starter short is acceptable when it is the clearest candidate and price remains below the fast and slow 5m SMA context.
- If `candidate_context.reasons` includes `short_setup_pre_breakdown_watch` or `long_setup_pre_breakout_watch`, treat that as an early continuation watchlist signal; slightly sub-average 5m volatility alone is not enough reason to force `hold`.
- When one ranked symbol has the clearest directional case, a small starter entry can be acceptable even if the setup is not perfect.
- If the snapshot market type is `spot`, prefer buy/close/hold and avoid sell unless the best outcome is effectively flattening a position.
- If the snapshot market type is `future`, then buy means long, sell means short, and close means flatten.
- If the chosen symbol already has an open position in the snapshot, or its `candidate_context.manage_only` flag is true, treat the decision as position management first rather than as a fresh entry search.
- For an existing position, hold is the default when the current side is still supported and there is no clear post-cost reason to flatten.
- For a fresh futures entry, keep `stop_loss_pct` positive and meaningful. `take_profit_pct` may be `0` when you prefer to let the position run without a fixed profit cap.
- Avoid very tight profit targets. When you do set one, `0.02` to `0.03` is usually stronger than a tiny scalp target when momentum and higher-timeframe bias are aligned.
- If the latest closed bar already extends far in the proposed trade direction and looks climactic or terminal, prefer `hold` rather than chasing it even when participation is expanding aggressively.
- For fresh futures shorts, do not chase continuation breakdowns when local RSI is already washed out and the latest closed bar is still pinned near the low; prefer waiting for a failed rebound or a cleaner re-break.
- For fresh futures longs, do not immediately re-enter the same long after an adverse loss-cut unless several closed 5m bars have clearly rebuilt support.
- For fresh futures longs, avoid chasing a weak reclaim or late breakout when the latest closed 5m bar is already stretched, local RSI is elevated, and volume has already expanded; prefer waiting for a cleaner pullback or sturdier rebuild.
- For fresh futures longs, if `return_24bars` is only marginally positive after costs and price is still at or below the 5m fast SMA context, do not let the unchanged 1h long bias or one rebound bar talk you into an entry; prefer `hold` until the reclaim is cleaner.
- For fresh futures rebound or continuation longs, require either RSI to have recovered back above roughly neutral with supportive SMA context, or clearly stronger multi-bar follow-through. A single strong rebound bar from sub-50 RSI with only modest participation is usually a wait, not a buy.
- For an existing long, prefer close only when the latest closed bar, fast/slow SMA context, or higher-timeframe bias have turned against the long strongly enough that avoiding further adverse move is likely worth the close cost.
- For an existing short, prefer close only when the latest closed bar, fast/slow SMA context, or higher-timeframe bias have turned against the short strongly enough that avoiding further adverse move is likely worth the close cost.
- If an existing position has already built a meaningful unrealized gain, prefer close rather than letting that gain retrace materially unless the latest closed bars still show clearly strengthening follow-through in the same direction.
- If an existing futures position spends several closed 5m bars drifting against the trade and unrealized PnL is only small or already negative, do not rely on the unchanged 1h bias alone; prefer `close` over repeated `hold` once near-term structure clearly weakens.
- For an existing starter long opened on a rebound or continuation idea, if 2 closed 5m bars pass without upside follow-through and unrealized PnL is still small or negative, prefer `close` over another passive `hold` even when the 1h bias is unchanged.
- For a new futures short, prefer sell when downside evidence is aligned: higher-timeframe bias is short or clearly bearish, `return_24bars` is not fighting the short thesis, fast/slow SMA context is supportive, the latest closed bar is not a clear rebound, and participation is at least adequate.
- A `flat` higher-timeframe bias can still allow a fresh short, but only when the 5m breakdown is clean, not terminal, and the post-cost edge is clearer than `hold`.
- If the latest closed bar strongly rebounds against the short thesis, or if the setup mostly relies on a vague "still bearish overall" story, prefer hold.
- Use hold when multiple core signals conflict, when there is no clear best symbol, or when waiting is still clearly better than acting.

Do not add markdown. Do not add explanations outside JSON.
